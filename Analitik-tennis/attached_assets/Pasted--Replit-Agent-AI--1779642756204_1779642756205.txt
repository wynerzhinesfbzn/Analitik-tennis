🎾 Промт для создания «Теннисного аналитика с AI-совещанием»

Цель:

Создать веб-приложение, которое работает как профессиональный теннисный аналитик.
Пользователь загружает скриншот с линией букмекера (коэффициенты, события) или вводит названия игроков/турнира.
Приложение анализирует все доступные данные (статистика, форма, покрытие, последние матчи, личные встречи, движение линии) и выдаёт точный прогноз в формате совещания 3-х AI-агентов (каждый со своей ролью).
Результат: конкретные ставки (включая экспрессы и системы), рекомендуемые коэффициенты, стратегия и процент банка.

Технические требования (Replit):

· Язык: Python + Flask/FastAPI
· UI: Streamlit или Gradio (просто для загрузки скриншотов и вывода)
· Анализ изображений: GPT-4o API (vision) или Gemini 1.5 Pro (чтобы читать коэффициенты и события со скриншота)
· Агенты: 3 вызова LLM (GPT-4o или Claude) с разными системными промптами (см. ниже)
· Поддержка аудио: запись вопроса пользователя → Whisper STT → ответ агентов → TTS (например, ElevenLabs или gTTS) в виде общего подкаста
· Живые данные: интеграция с API (Tennis Data API, Bet365 scrape через RapidAPI или парсинг flashscore.com через requests + BeautifulSoup) — при отсутствии ключа использовать локальную базу последних 100 топ-матчей
· Точность: каждый агент обязан запрашивать реальные актуальные данные (форма за 12 месяцев, х2х, покрытие, погода, травмы, турнирная мотивация)

Роли AI-агентов (всегда обсуждают в формате диалога):

1. Аналитик статистики (Stats Expert)
   · Сравнивает подачу, приём, брейк-пойнты, выигранные мячи на первой подаче, количество двойных ошибок, тай-брейки.
   · Выводит объективные вероятности на каждый исход (П1, П2, тотал, гандикап).
   · Формулирует: «По цифрам я вижу, что…»
2. Беттинг-стратег (Odds Strategist)
   · Сравнивает расчётную вероятность с коэффициентами букмекера (со скриншота).
   · Ищет валуй (value bet).
   · Предлагает конкретные рынки: фора, тотал матча/сетов, точный счёт по сетам, ставка на возвращение после тай-брейка.
   · Указывает, какой размер банка ставить (1-5%).
   · Может собрать экспресс из 2-3 матчей (если есть несколько событий на скрине).
3. Контекстный эксперт (Tennis Context)
   · Учитывает психологию: переход с грунта на траву, усталость после длинного матча, личные встречи на этом покрытии, проблемы с тренером, травмы (даже скрытые).
   · Знает всех игроков Топ-150 и середняков Челленджера.
   · Делает итоговый вердикт: «ставка надёжна/рискована, рекомендую фора/тотал/исход».

Как происходит «совещание» (единый вывод для пользователя):

Приложение вызывает всех трёх агентов последовательно, но в ответе формирует живой диалог, например:

🎙 Stats Expert: «У Медведева 72% выигранных мячей на первой подаче на харде. Синнер на приёме – 34% выигранных розыгрышей. Модель даёт вероятность П2 = 61% при кф 1.85. Это валуй, но тотал больше 22.5 выглядит ещё надёжнее – 78% подобных матчей уходят на 3 сета.»

💬 Betting Strategist: «Согласен по тоталу. На скрине БК даёт ТБ 22.5 за 1.90 – играем. Экспресс: ТБ в этом матче + фора (+2.5) на аутсайдера в соседней игре (Рублёв vs Алькарас) – суммарный кф 3.20, ставим 2% банка.»

🧠 Context Expert: «Но осторожно: Синнер играл вчера 3 часа, а Медведев отдыхал 2 дня. Усталость снизит качество приёма. Поэтому моя рекомендация – фора Синнера (-1.5) по геймам, кф 1.92. Экспресс не советую из-за риска быстрой победы фаворита.»

Итог от системы: три чётких варианта ставок с кф и % банка + экспресс (если есть) + страховка (например, возврат ставки по кэшауту).

Обработка скриншотов:

· Пользователь загружает изображение (линия БК).
· Приложение распознаёт текст (OCR или GPT Vision) и определяет:
  · названия игроков
  · коэффициенты на П1, П2, форы, тоталы
  · турнир, дату
· Далее агенты дополняют эти данные из внешних источников.

Альтернативный ручной ввод:

Если скриншота нет – пользователь пишет: «Джокович – Циципас, Шанхай, хард».
Аналитик сам подтягивает актуальную линию (через API или парсинг) и выдаёт прогноз.

Аудио-обсуждение (опция):

· Пользователь нажимает «Задать вопрос голосом» → микрофон → приложение отправляет аудио в Whisper → текст идёт к агентам.
· Ответ агентов озвучивается (голосом разного тембра для каждого эксперта) в виде короткого аудио-файла или прямого потока.
· Можно скачать аудио-обсуждение как «подкаст прогноза».

Формат финального вывода:

```
🔮 Теннисный прогноз  
Матч: [Игрок 1] – [Игрок 2], турнир, покрытие, дата  

📊 Анализ: [2-3 абзаца статистики]  

🤝 Совещание агентов (диалог)  

✅ Лучшие ставки (от наиболее вероятных к рискованным):
1. Исход / Тотал / Фора – кф X.XX (размер ставки Y% банка)
2. ...  
3. Экспресс дня: событие 1 (кф), событие 2 (кф) – общий кф Z.ZZ (ставка 1-2%)

⚠️ Риски: [что может пойти не так]  
📉 Кэшаут: при счёте X:Y лучше зафиксировать прибыль
```

Требования к надёжности:

· Если агенты расходятся во мнении – система показывает голосование (2 против 1) и ставит пометку «Спорная ставка → фантомный бет (не более 1%)».
· Каждый прогноз сохраняется в БД (SQLite) для последующей проверки точности.
· Должна быть страница «История точности» (плюс/минус по реальным результатам).

Как использовать этот промт в Replit:

1. Открой Replit → Create Repl → выбрать AI Agent.
2. Вставь этот промт в поле «Describe your app».
3. Нажми Generate — он сам создаст все файлы, установит зависимости (OpenAI, gradio, whisper, beautifulsoup, etc.).
4. Добавь свои API-ключи (OpenAI, RapidAPI для тенниса) в Secrets.
5. Запусти — сервис готов.

Если Replit Agent не справится с мультиагентностью, используй этот промт как техническое задание и отдай его в Claude или GPT Engineer внутри Replit. Но Agent должен потянуть.
